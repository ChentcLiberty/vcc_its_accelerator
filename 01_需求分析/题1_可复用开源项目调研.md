# 题1 可复用开源项目调研

更新时间：2026-05-19

## 结论

当前最值得复用的不是现成 `RTL`，而是：

1. `VVC` 高质量软件参考实现
2. 可直接抽取为黄金模型的逆变换代码
3. 验证基础设施

当前没有找到一个成熟、活跃、可直接改成比赛主工程的 `VVC ITS / LFNST Verilog RTL` 开源仓库。  
因此推荐路线是：

- `软件参考复用`
- `硬件 RTL 自主实现`

## 推荐复用优先级

### 1. VVdeC

- 仓库：`fraunhoferhhi/vvdec`
- 链接：<https://github.com/fraunhoferhhi/vvdec>
- 价值判断：`最高`

原因：

- 仓库主页说明它是 Fraunhofer 的 `H.266/VVC decoder implementation`，并且是 `BSD-3-Clause-Clear` 许可。
- 代码树里有 `source/` 和 `tests/`，工程质量较高，维护也活跃。
- 对题1最关键的是其逆变换实现路径。

最值得抽取的文件：

- `source/Lib/CommonLib/TrQuant.cpp`

关键原因：

- 文件中有 `fastInvTrans[NUM_TRANS_TYPE][g_numTransformMatrixSizes]`
- 能直接看到：
  - `fastInverseDCT2_B2/B4/B8/B16/B32/B64`
  - `fastInverseDCT8_B4/B8/B16/B32`
  - `fastInverseDST7_B4/B8/B16/B32`
- 也包含 `TrQuant::xInvLfnst(...)`

推荐复用方式：

- 抽它的：
  - `变换类型选择`
  - `块大小映射`
  - `LFNST 调用条件`
  - `scan / top-left low-frequency region` 处理
- 不直接照搬 C++ 结构到 RTL
- 优先把它转成 `Python/NumPy golden model`

结论：

- `VVdeC` 是题1最该复用的软件黄金参考

### 2. VTM 参考软件镜像

- 仓库：`OpenVVC/VVCSoftware_VTM`
- 链接：<https://github.com/OpenVVC/VVCSoftware_VTM>
- 价值判断：`高，但次于 VVdeC`

原因：

- 仓库 About 明确写的是 `Fork of VTM VVC Software from Franhauffer HHI`
- 它更接近标准参考实现
- 适合作为行为正确性的最终对照

最值得抽取的文件：

- `source/Lib/CommonLib/TrQuant.cpp`

推荐复用方式：

- 用来核对：
  - `LFNST` 行为
  - `DCT2/DCT8/DST7` 逆变换路径
  - 模式切换和边界条件
- 不建议把它当成主开发底座

结论：

- `VTM` 更适合作为 `oracle`，不适合直接当工程骨架

### 3. uvg266

- 仓库：`ultravideo/uvg266`
- 链接：<https://github.com/ultravideo/uvg266>
- 价值判断：`中高`

原因：

- 仓库主页标明是 `An open-source VVC encoder`
- 许可为 `BSD-3-Clause`
- README 里明确写了测试中 `Bitstream validity is checked with VTM`

对题1的价值不在于 RTL，而在于：

- 生成和补充 `VVC` 测试样例
- 为 `LFNST / MTS` 场景补充软件侧输入
- 做交叉验证

结论：

- `uvg266` 适合当测试样例辅助源，不适合当逆变换主参考实现

### 4. cocotb

- 仓库：`cocotb/cocotb`
- 链接：<https://github.com/cocotb/cocotb>
- 价值判断：`中高`

原因：

- 仓库主页 About 直接写的是 `Python-based chip (RTL) verification`
- 和当前题1路线非常匹配：
  - `Python/NumPy golden model`
  - `RTL 对拍`
  - directed case + regression

结论：

- 如果题1验证要快速起步，`cocotb` 很适合直接复用

### 5. fplib

- 仓库：`SkyworksSolutionsInc/fplib`
- 链接：<https://github.com/SkyworksSolutionsInc/fplib>
- 价值判断：`中`

原因：

- 仓库主页 About 写的是 `Fixed point math library for SystemVerilog`
- 适合用于：
  - 定点算术试验
  - 中间位宽管理
  - 小规模固定点验证

局限：

- 不是 VVC 专用库
- 如果最后坚持纯 `Verilog` 交付，复用价值会降低

结论：

- 可做辅助，不应成为主依赖

## 当前不建议投入太多时间找什么

不建议继续花大量时间找下面这类仓库：

- 现成 `VVC LFNST Verilog RTL`
- 可直接综合上板的 `ITS` 硬件开源 IP

当前公开高质量项目里，这类资源非常少；继续深挖，时间收益比不高。

## 当前最推荐的复用组合

推荐组合：

1. `VVdeC`：主黄金参考
2. `VTM`：标准行为对照
3. `uvg266`：补测试样例
4. `cocotb`：验证框架
5. `自写 RTL`：`LFNST core + 1D inverse transform + transpose buffer + scheduler`

## 对题1的直接动作建议

下一步不是再搜更多仓库，而是：

1. 从 `VVdeC / VTM` 抽 `LFNST` 和 `inverse transform` 关键路径
2. 落成 `Python/NumPy` 参考模型
3. 先写 `LFNST RTL`
4. 再上 `1D DCT2/DCT8/DST7`
