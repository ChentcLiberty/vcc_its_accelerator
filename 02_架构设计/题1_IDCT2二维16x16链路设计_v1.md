# 题1 IDCT2 二维16x16链路设计 v1

## 目标

在现有：

- `IDCT2 1D`
- `transpose buffer`
- `8x8` 二维 `IDCT2`

基础上，再向更大块型推进一层：

- [idct2_2d16_core.v](../05_rtl/idct2_2d16_core.v)

## 这版覆盖什么

当前只覆盖：

- `16x16`
- 横纵两个方向都固定 `IDCT2`
- 非 `LFNST` 路径

当前不覆盖：

- `DCT8 / DST7`
- `32/64`
- 稀疏输入装载顶层
- 最终 `10-bit` 规范化输出

## 当前数据路径

### 第一阶段：行向 `IDCT2`

逐行处理 `16` 个输入点，每行结果按：

- `4点/拍`

写入中间 buffer。

### 第二阶段：转置读列向量

借助：

- `rd_transpose = 1`

把中间 buffer 中某一列按 `4+4+4+4` 的四段读出来，拼成一个完整 `16` 点列向量。

### 第三阶段：列向 `IDCT2`

对上一步拼好的 `16` 点列向量再跑一次 `1D IDCT2`。

### 第四阶段：转置写最终矩阵

列向 `IDCT2` 的结果用：

- `wr_transpose = 1`

写回最终 buffer，使最终 buffer 内部保持行优先结果矩阵。

### 第五阶段：行优先流出

最后从最终 buffer 中按行优先、`4点/拍` 输出，总计：

- `64` 组输出

## 和 `8x8` 版本的关键差异

`16x16` 版本最明显的新点是：

- 一列向量不再是两段 `4+4`
- 而是四段 `4+4+4+4`

所以状态机里新增了四个列装载阶段：

- `PREP_COL_0`
- `PREP_COL_1`
- `PREP_COL_2`
- `PREP_COL_3`

## 当前意义

这一步把二维 `IDCT2` 真正从：

- 小规模 `8x8`

推进到了：

- 更接近题面主战场的 `16x16`

后面如果继续往：

- `32x32`
- `DCT8 / DST7`

推，主要变化就会集中在：

- 有效长度
- 中间位宽
- 状态机分段数

而不会再是完全换一套思路。
