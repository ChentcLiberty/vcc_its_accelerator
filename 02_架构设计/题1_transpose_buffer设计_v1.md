# 题1 transpose buffer 设计 v1

## 目标

把 `ITS` 二维链路里“列变换结果缓存 + 读视图切换”这件事单独抽成一个可综合模块：

- [its_transpose_buffer.v](../05_rtl/its_transpose_buffer.v)

这一步对应题1整体微架构里的：

- `its_transpose_buffer`

## 这一步为什么值得单独做

前面的 `4x4` 二维闭环虽然已经能跑通，但中间矩阵还是直接塞在局部寄存器阵列里。

这样的问题是：

- 中间缓存逻辑绑死在某一条链路里
- 后面扩到更大块型时，不利于复用
- 没法单独验证“普通读视图 / 转置读视图”

所以这一步先把它抽成独立模块，再让二维链路去复用它。

## 当前接口

### 写侧

- `n_rows / n_cols`
- `wr_valid`
- `wr_row_idx`
- `wr_col_base`
- `wr_data_0..3`

当前写侧约定是：

- 一次写入同一行中连续 `4` 个元素
- 默认是“行优先写入”

### 读侧

- `rd_transpose`
- `rd_major_idx`
- `rd_minor_base`
- `rd_data_0..3`

读侧支持两种视图：

1. `rd_transpose = 0`
   - 普通行视图
   - 读 `row = rd_major_idx`，列从 `rd_minor_base` 开始的连续 `4` 个值
2. `rd_transpose = 1`
   - 转置视图
   - 读转置矩阵中 `row = rd_major_idx`，列从 `rd_minor_base` 开始的连续 `4` 个值

## 当前实现

这版先用：

- `reg array`
- 组合读
- 同步写

来把功能收稳。

底层索引统一按：

`mem[row * MAX_DIM + col]`

存储，`MAX_DIM` 当前默认设成 `64`，覆盖题面最大块型。

## 边界处理

读写时都带边界判断：

- 行号越界不访问
- 列号越界不访问
- 越界读返回 `0`

这点是刻意做的，因为后面更大块型和分组输出时，最后一组可能不一定总是完整 `4` 点。

## 当前在二维链路里的用法

在 [lfnst_idct2_2d4_core.v](../05_rtl/lfnst_idct2_2d4_core.v) 里，这版先用它做：

- 列链路输出的中间矩阵缓存
- 行向 `IDCT2` 装载前的统一读口

当前二维 `4x4` 链路里还没有打开 `transpose` 视图，仍然先按普通行视图读出。

这么做是刻意的：

- 先把模块边界抽稳
- 再在更大块型里引入真正的“转置读写调度”

## 当前意义

这一步之后，题1的二维路径已经不再把“中间缓存”写死在链路内部，而是开始向：

- 通用中间 buffer
- 可切换读视图
- 更大块型复用

这条方向收敛。
