# 题1 LFNST到IDCT2列链路设计 v1

## 目标

这版不是完整 `ITS top`，而是先打通一个真实的中间子链路：

- `LFNST`
- `4x4` 回填
- `列向 IDCT2`

对应模块：

- [lfnst_idct2_col4_core.v](../05_rtl/lfnst_idct2_col4_core.v)

## 为什么先做这一步

直接跳完整 `LFNST -> 列变换 -> 转置 -> 行变换 -> 输出打包` 会把太多问题搅在一起：

- `LFNST` 输出回填
- `1D` 变换调用顺序
- 中间矩阵布局
- 输出反压
- 行列阶段的边界

所以这版先把最关键、也最容易出错的第一段收稳：

`LFNST vector -> 4x4 block -> per-column IDCT2`

## 当前边界

当前只支持：

- `4x4 TU`
- `IDCT2`
- 列向 `1D` 变换

当前还不覆盖：

- `8x8 / 16x16 / 32x32 / 64x64`
- `DCT8 / DST7`
- 转置 buffer
- 行向第二次 `1D` 变换
- 最终 `10-bit` 打包输出

## 数据路径

### 1. 输入锁存

模块起始输入仍然是 `x_bar[16]`，也就是 `LFNST` 所需的扫描向量。

这样做的原因是：

- 现阶段先复用已经稳定的 `lfnst_core`
- 不在这版里提前引入题面稀疏输入装载器

### 2. LFNST阶段

当 `lfnst_idx != 0` 时：

- 启动 [lfnst_core.v](../05_rtl/lfnst_core.v)
- 收满 `16` 个 `LFNST` 输出元素

当 `lfnst_idx == 0` 时：

- 不启动 `LFNST`
- 直接把输入 `x_bar[16]` 当作 bypass 向量

这样做的意义是：

- 同一条链路可以同时验证 `LFNST enabled` 和 `LFNST bypass`

### 3. 4x4 回填

`LFNST` 输出的 `16` 元素仍然是扫描向量，不是行优先 `4x4` 矩阵。

所以要先按固定扫描顺序回填成完整 `4x4 block`。

当前回填顺序和参考模型一致：

- `(0,0)`
- `(1,0)`
- `(0,1)`
- `(2,0)`
- `(1,1)`
- `(0,2)`
- `(3,0)`
- `(2,1)`
- `(1,2)`
- `(0,3)`
- `(3,1)`
- `(2,2)`
- `(1,3)`
- `(3,2)`
- `(2,3)`
- `(3,3)`

### 4. 列向 IDCT2

回填后的 `4x4 block` 进入列向 `IDCT2`：

- 一次处理一列
- 每列长度固定 `4`
- 复用现有 [idct2_1d_core.v](../05_rtl/idct2_1d_core.v)

调用方式是：

- `n_tbs = 4`
- `non_zero_size = 4`
- 每轮只把某一列的 `4` 个元素装到 `x_0..x_3`
- `x_4..x_63` 清零

### 5. 输出整理

四列都跑完后，模块内部已经得到一个新的 `4x4` 中间矩阵。

外部输出按行优先、`4点/拍` 流出：

- 第一拍：第 `0` 行
- 第二拍：第 `1` 行
- 第三拍：第 `2` 行
- 第四拍：第 `3` 行

## 控制状态

当前状态机分成几段：

1. `IDLE`
2. `LFNST bypass` 或 `LFNST launch/run`
3. `4x4 remap`
4. `per-column IDCT2 prepare/launch/run`
5. `row stream out`

这版故意没有追求流水重叠，而是优先把阶段边界做清楚。

## 当前实现价值

这版的核心价值不是性能，而是把下面三个接口关系钉死：

- `LFNST vector` 和 `4x4 TU` 的映射关系
- `4x4 TU` 和 `1D IDCT2` 列输入的关系
- 列变换中间结果如何重新按行优先输出

这些关系一旦错，后面接 `transpose / row transform / ITS top` 都会错。
