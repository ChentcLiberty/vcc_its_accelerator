# 题1 LFNST到IDCT2二维4x4链路设计 v1

## 目标

这版把 `4x4` 的二维反变换链路真正串起来：

- `LFNST`
- 列向 `IDCT2`
- 行向 `IDCT2`

对应模块：

- [lfnst_idct2_2d4_core.v](../05_rtl/lfnst_idct2_2d4_core.v)

## 为什么这一步有价值

前一版 [lfnst_idct2_col4_core.v](../05_rtl/lfnst_idct2_col4_core.v) 只打通了：

- `LFNST`
- `4x4` 回填
- 列向 `IDCT2`

这一步再往前推一层，形成第一个真正“接近完整 `ITS`”的二维闭环：

`LFNST -> 列向 IDCT2 -> 行向 IDCT2`

虽然当前只支持 `4x4`，但验证意义已经明显高于单条列链路。

## 当前边界

当前只支持：

- `4x4 TU`
- 横纵两个方向都固定 `IDCT2`
- `LFNST bypass / enabled`

当前还不覆盖：

- `DCT8 / DST7`
- `8/16/32/64`
- 通用 `transpose buffer`
- 最终规范化和 `10-bit` 打包输出

## 当前数据路径

### 1. 输入锁存

模块输入仍然是 `x_bar[16]` 扫描向量。

### 2. 列链路复用

第一阶段不再重复实现 `LFNST + 列向 IDCT2`，而是直接复用：

- [lfnst_idct2_col4_core.v](../05_rtl/lfnst_idct2_col4_core.v)

这样做的好处：

- 避免重复写状态机
- 直接复用已经通过 directed + UVM 的子链路

### 3. 中间矩阵缓存

列链路输出的是按行优先、`4点/拍` 的 `4x4` 中间矩阵。

当前版本已经把这层中间缓存抽成：

- [its_transpose_buffer.v](../05_rtl/its_transpose_buffer.v)

这次的目的不是立刻在 `4x4` 链路里用到复杂转置调度，而是先把：

- 行优先写入
- 独立中间缓存
- 后续可切换转置读视图

这几个边界从二维链路里拆出来。

### 4. 行向 IDCT2

第二阶段再次复用 [idct2_1d_core.v](../05_rtl/idct2_1d_core.v)：

- 一次处理一行
- 长度固定 `4`
- 四行依次发给 `1D IDCT2`

与前一版不同的是：

- 列链路输出是 `32-bit`
- 所以行向 `IDCT2` 的输入位宽提升到 `32-bit`
- 行向累加位宽也同步放大，避免中间二维结果溢出

### 5. 最终输出

最终 `4x4` 二维结果按行优先、`4点/拍` 输出：

- 第 `0` 行
- 第 `1` 行
- 第 `2` 行
- 第 `3` 行

## 当前状态机

这版状态分成：

1. `IDLE`
2. `START_COL`
3. `RUN_COL`
4. `PREP_ROW`
5. `START_ROW`
6. `RUN_ROW`
7. `STREAM_OUT`

目前仍然是功能优先、阶段串行，不追求重叠流水。

## 当前意义

这版真正钉住了三件后面非常关键的事：

1. `x_bar -> 2D` 的第一条完整路径
2. 列结果到行结果的中间位宽关系
3. 二维阶段结束后的行优先输出格式

后面继续做：

- 更大块型
- `DCT8 / DST7`
- 真正依赖转置视图的调度路径

都可以在这版二维闭环之上扩。
