# 题1 IDCT2 二维8x8链路设计 v1

## 目标

在现有：

- `IDCT2 1D`
- `transpose buffer`

基础上，补第一版更大块型二维路径：

- [idct2_2d8_core.v](../05_rtl/idct2_2d8_core.v)

## 这版覆盖什么

当前只覆盖：

- `8x8`
- 横纵两个方向都固定 `IDCT2`
- 非 `LFNST` 路径

这版不覆盖：

- `DCT8 / DST7`
- `16/32/64`
- 稀疏输入装载
- 最终 `10-bit` 规范化输出

## 当前数据路径

### 第一阶段：行向 IDCT2

先对每一行跑一次 `1D IDCT2`。

这一步的好处是：

- `1D` 输出天然按行优先给出
- 正好适合直接写进 `transpose buffer`

### 第二阶段：转置读列向量

第一阶段结果写入中间 buffer 后，再通过：

- `rd_transpose = 1`

把某一列读成一个连续 `8` 点向量。

### 第三阶段：列向 IDCT2

对转置读出的列向量再次跑 `1D IDCT2`。

### 第四阶段：转置写最终矩阵

列向 `IDCT2` 的结果本质上还是“某一列”的 `8` 个点，所以这次通过：

- `wr_transpose = 1`

把它写回最终矩阵 buffer。

这样最终 buffer 内部保持的是：

- 行优先的最终 `8x8` 结果

### 第五阶段：按 `4点/拍` 流出

最后按行优先、`4点/拍` 从最终 buffer 流出。

## 当前结构取舍

这版没有强行做：

- 双 pass 重叠流水
- 多个 `1D IDCT2` 并行
- 行列阶段 overlap

而是先用串行状态机把功能闭环收稳。

## 当前意义

这一步之后，题1已经不再只停留在 `4x4` 的概念闭环，而是开始进入：

- 更大块型
- 双 buffer
- 转置读写都参与数据流

这条真正接近通用二维变换核的路径。
