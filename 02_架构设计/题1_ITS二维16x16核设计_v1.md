# 题1 ITS二维16x16核设计 v1

## 目标

在现有：

- `ITS 1D`
- `transpose buffer`
- `ITS 2D 8x8`

基础上，把共享二维壳子继续往更大块型推进一层：

- [its_2d16_core.v](../05_rtl/its_2d16_core.v)

## 这版覆盖什么

当前只覆盖：

- `16x16`
- 横纵两个方向都固定 `DCT8`
- 非 `LFNST` 路径

当前不覆盖：

- `DST7 16x16`
- `32/64`
- 顶层稀疏输入装载协议
- 本轮 directed 中的单拍 `out_req` 反压检查
- 最终 `10-bit` 规范化输出

## 当前数据路径

### 第一阶段：行向 `ITS 1D`

逐行处理 `16` 个输入点，每行结果按：

- `4点/拍`

写入中间 buffer。

### 第二阶段：转置读列向量

借助：

- `rd_transpose = 1`

把中间 buffer 中某一列按 `4+4+4+4` 的四段读出来，拼成完整 `16` 点列向量。

### 第三阶段：列向 `ITS 1D`

对上一步拼好的 `16` 点列向量再跑一次 `1D DCT8`。

### 第四阶段：转置写最终矩阵

列向结果用：

- `wr_transpose = 1`

写回最终 buffer，使最终 buffer 内部保持行优先结果矩阵。

### 第五阶段：行优先流出

最后从最终 buffer 中按行优先、`4点/拍` 输出，总计：

- `64` 组输出

## 和旧 `IDCT2 16x16` 版本的关系

这版不是重新发明一套 `16x16` 状态机，而是复用旧的 `16x16` 二维调度思路，把底层 `1D` 核切到共享的：

- [its_1d_core.v](../05_rtl/its_1d_core.v)

核心变化只有两类：

1. 增加 `ROW_TR_TYPE / COL_TR_TYPE`
2. ROM 切到共享 `its_1d_tables.memh`

这样后面往：

- `DST7 16x16`
- 更统一的 `ITS 2D 16x16`

推进时，不需要再推倒重来。

## 当前意义

做到这一步后，项目里已经有两层共享二维壳子：

- `ITS 2D 8x8`
- `ITS 2D 16x16`

这比继续堆专用 `IDCT2` 版本更贴近题目最后要交的“可复用 ITS 微架构”。
