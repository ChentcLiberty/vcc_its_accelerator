# 题1 官方接口顶层第一版验证说明 v1

## 验证对象

- [its_top_official_if_stage1.v](../05_rtl/its_top_official_if_stage1.v)

## 这版做什么

这版不是再验证单个二维核，而是第一次验证“题面官方接口顶层”本身，包括：

1. `it_info` 解码
2. `it_data_in_req` 驱动
3. sparse 输入按 `it_data_addr` 恢复到内部块
4. `4x4` 路径的 `raster -> xBar` 映射
5. `4点/拍` 的 `40-bit` 打包输出
6. `it_done` 收尾

## 当前 case

1. `4x4 DCT2 + LFNST`
   - `tu_width = 4`
   - `tu_height = 4`
   - `tr_type_hor = DCT2`
   - `tr_type_ver = DCT2`
   - `lfnst_idx = 1`
   - `16` 个点按光栅地址送入官方顶层
   - 参考结果来自 [lfnst_idct2_2d4_core.v](../05_rtl/lfnst_idct2_2d4_core.v)
2. `8x8 DCT8`
   - `tu_width = 8`
   - `tu_height = 8`
   - `tr_type_hor = DCT8`
   - `tr_type_ver = DCT8`
   - `lfnst_idx = 0`
   - `64` 个点按光栅地址送入官方顶层
   - 参考结果来自 [its_2d8_core.v](../05_rtl/its_2d8_core.v)

## 参考策略

这版 smoke testbench 没有再手抄一套期望矩阵，而是直接把：

- `4x4 LFNST + IDCT2`
- `8x8 DCT8`

两条现有已通过的子链路模块，作为同拍参考模型实例化在 testbench 里，再把官方顶层的 `40-bit` 打包输出和参考路径经 `10-bit` 饱和后的打包结果逐拍比对。

这样这版更像“接口适配验证”，而不是再次重复验证内核数学本身。


## 当前边界

这版 smoke 还不是完整赛题顶层回归，只覆盖：

- `4x4 DCT2 + LFNST`
- `8x8 DCT8`

还没有覆盖：

- `8x8 DCT2`
- `8x8 DST7`
- `16x16` 三条路径
- unsupported 模式
- `it_data_out_req` 反压场景

这些后面应该继续逐项补齐。
