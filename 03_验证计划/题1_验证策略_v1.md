# 题1 验证策略 v1

## 当前原则

第一阶段不一上来堆完整 `UVM`，先把 golden model 收稳；在 golden model 和 directed self-check 可靠之后，再补轻量 `UVM` 骨架。

当前已经具备的基础是：

- `LFNST` Python model + RTL self-check
- `IDCT2` Python model + RTL self-check

所以现在的推荐做法是两层并行：

- `Python / NumPy` 参考模型
- `SystemVerilog` directed 自检 testbench
- 轻量 `UVM` block-level 环境
- 模块级与顶层级双层闭环

原因：

- 没有 golden model 时，方法学再重也没有意义
- 有了稳定 golden 之后，`UVM` 才能真正复用到多模块和联调

## 参考模型拆分

建议先做四块：

1. `lfnst_ref.py`
   - 支持 `nTrs = 16 / 48`
   - 支持不同 `lfnstTrSetIdx / lfnst_idx`
2. `idct2_ref.py`
3. `idct8_ref.py`
4. `idst7_ref.py`

最终再封成：

- `its_ref.py`

## 模块级验证

### `lfnst_core`

需要检查：

- `nTrs = 16 / 48`
- `lfnstTrSetIdx`
- `lfnst_idx`
- 限幅逻辑
- 左上非零区域之外是否正确清零

### `its_1d_core`

需要检查：

- `DCT2 / DCT8 / DST7`
- 不同长度 `4 / 8 / 16 / 32 / 64`
- 边界值、全零、单脉冲、随机输入

### `transpose_buffer`

需要检查：

- 写入顺序
- 转置读出顺序
- 边界块大小

## 顶层级验证

顶层要检查三件事：

1. 输入装载是否按题面协议恢复到内部块
2. `LFNST -> 列变换 -> 行变换 -> 输出整理` 是否全链一致
3. 输出是否按 `4点/拍` 光栅顺序给出

## 第一阶段推荐验证方案

### 推荐方案

- 参考模型：`Python/NumPy`
- RTL 仿真：`VCS`
- 第一层：轻量 `SV testbench`
- 第二层：轻量 `UVM`

当前已经开始的 UVM 落地：

- `IDCT2 1D` block-level UVM 骨架
- 已通过 `5` 组 smoke case 和 `1` 组单拍输出反压场景
- `LFNST -> 列向 IDCT2` 子链路 UVM 骨架
- 已通过 `bypass/enabled` 两组 smoke case 和 `1` 组单拍输出反压场景
- 包括：
  - `driver`
  - `monitor`
  - `scoreboard`
  - `env`
  - `test`
  - `interface assertion`

当前已经完成的 directed 二维闭环：

- `4x4 LFNST -> 列向 IDCT2 -> 行向 IDCT2`
- 已通过 `bypass` 和 `LFNST enabled` 两组 case

### 当前不建议优先做的事

- 一上来搭大而全的 SoC 级 `UVM`
- 一上来做全模式随机回归
- 在 block-level UVM 还没收稳前，过早堆太多覆盖率模型

## 第一周验证交付目标

至少做出：

1. `LFNST` Python 参考模型
2. `LFNST` RTL 自检
3. `LFNST` 覆盖 `nTrs = 16 / 48` 的 directed case
4. 一份能稳定复现的对拍脚本
