# 题1 第一周开工清单

## 第一周目标

不是做完整 `ITS`，而是拿下三个东西：

1. 官方矩阵资料落地成机器可读格式
2. `LFNST` 参考模型跑通
3. `LFNST` RTL + 自检验证跑通

## Day 1

- 整理官方题面和本地附件
- 确认：
  - `nTrs = 16 / 48`
  - `lfnstTrSetIdx`
  - `lfnst_idx`
  - 输入输出低频区域约束
- 建好比赛目录结构

## Day 2

- 把 `LFNST` 系数矩阵转成机器可读表
- 优先格式：
  - `csv`
  - 或 Python list / JSON
- 先只保证可读和可校验，不追求优雅

## Day 3

- 写 `lfnst_ref.py`
- 支持：
  - `nTrs = 16`
  - `nTrs = 48`
  - clip
  - 左上有效区规则

## Day 4

- 拿几组固定样例对拍
- 形成：
  - 输入块
  - 预期输出块
  - 简单日志

## Day 5

- 写 `lfnst_core.v`
- 第一版只追求功能正确
- 系数可以直接 ROM 化

## Day 6

- 写 `tb_lfnst_core.sv` 或 `cocotb` 自检
- 做 directed case：
  - 全零
  - 单点激励
  - 随机低频块
  - `nTrs = 16 / 48`

## Day 7

- 收敛 bug
- 形成第一周交付：
  - `lfnst_ref.py`
  - `lfnst_core.v`
  - 自检 testbench
  - 结果截图 / 日志

## 第一周完成标准

满足以下四条才算完成：

1. 参考模型稳定输出
2. RTL 对拍通过
3. `nTrs = 16 / 48` 都覆盖到
4. 目录和文件命名已经具备继续扩展的基础

## 第一周之后的顺序

1. `IDCT2`
2. `DCT8 / DST7`
3. `2D top`
4. `4点/拍` 和时序优化
